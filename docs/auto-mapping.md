# Auto-Mapping Draft System

## What it does
`tools/auto-map-pdf.js` generates a first-draft mapping JSON from a court PDF in:

`assets/forms/ohio/{county}/{courtType}/{formType}.pdf`

Output:

`templates/ohio/{countySlug}/{courtType}/{formType}.draft.json`

Each generated field is marked:
- `autoGenerated: true`
- `needsReview: true`
- `confidence: 0.0 - 1.0`

## Run it

```bash
node tools/auto-map-pdf.js --county=wood --courtType=court-of-common-pleas --formType=sealing-conviction
```

## Detection behavior
- Extracts text items and coordinates with `pdfjs-dist`.
- Searches for common court labels (Case No., Judge, Name, Address, Charge, Offense Code, Signature, etc.).
- Places field boxes to the right of detected labels when available.
- Uses date type for `*date*` fields and text for the rest.

## Admin UI changes
- `admin/asset-manager.html` now shows mapping status:
  - No mapping
  - Draft mapping needs review
  - Mapped
  - Active
- Added actions:
  - **Generate Draft Mapping** (copies runnable CLI command)
  - **Open Draft in Mapper**

## Mapper workflow
- Open `admin/pdf-mapper.html` with `draft` and `final` query params.
- Load and edit `.draft.json`.
- Save final reviewed mapping as `.json` (download and place under `templates/.../{formType}.json`).

## Safety
- Drafts never auto-activate.
- Existing manual mappings remain untouched.
